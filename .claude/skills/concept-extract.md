# concept-extract — 概念抽出 Skill

物語化の前処理として、原典テキストのキーコンセプトを整理する。
長文・複雑な章を `/narrativize` に渡す前に使う。

## 使い方

```
/concept-extract
[原典テキストをここに貼り付け]
```

---

## 出力フォーマット

```
## 概念抽出レポート

### 📌 中心テーマ
[この章・節が最も伝えたいこと・1文]

### 🔑 主要概念（3〜7個）
1. **[概念名]**: [一言説明]
2. **[概念名]**: [一言説明]
...

### ❓ 初心者が持つ疑問（想定）
- [疑問1]
- [疑問2]
- [疑問3]

### 💡 比喩・たとえ話の候補
- [概念名] → [たとえ話のアイデア]
- [概念名] → [たとえ話のアイデア]

### 📖 物語化の推奨構成
- **弟子の最初の問い**: [推奨する導入の疑問]
- **クライマックス**: [対話の山場となるべきポイント]
- **核心の一文**: [「この話の核心」欄に入れるべき文]

### ⚠️ 注意点
[原典の意味を損なわないための注意事項]
```

---

## 処理手順

1. テキスト全体を読み、著者が最も強調している点を特定する
2. 専門用語・固有概念をリストアップする
3. 初心者が「なぜ？」「どういうこと？」と感じそうな箇所を特定する
4. 各概念に対応する日常的なたとえ話を考える
5. 物語化の流れを提案する
