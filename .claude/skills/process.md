# process — 統合物語化 Skill

テキストを貼り付けるだけで、概念抽出 → 分割判断 → 物語化 → 品質チェック → MD保存まで自動で完結する。

## 使い方

```
/process
[原典テキストをここに貼り付け]
```

オプション：
- `--source "書籍名"` — 原典の書籍名（省略時はAIが推定）
- `--slug "ファイル名prefix"` — 保存ファイル名の接頭辞（例: `genri-kouron`）

---

## 処理フロー

```
入力テキスト
    ↓
[STEP 1] 内部で concept-extract を実行（ユーザーへの表示は要点のみ）
    ↓
[STEP 2] 長さ・複雑さを判断
    ├─ 短い（1〜2節、概念3個以下）→ そのまま1話として処理
    └─ 長い（複数節、概念4個以上）→ chapter-split を実行
                                          ↓
                                    分割案をユーザーに提示
                                          ↓
                                    ✋ 確認待ち（「このまま進めてください」で続行）
    ↓
[STEP 3] 各話について narrativize を実行（複数話の場合は順番に）
    ↓
[STEP 4] 内部で story-review を実行（問題があれば自動修正してから保存）
    ↓
[STEP 5] content/stories/ に MDファイルを自動保存
    ↓
完了レポートを表示
```

---

## 長さ・分割の判断基準（AI自動判断）

| 条件 | 判断 | 処理 |
|------|------|------|
| 500字未満 or 概念1〜2個 | 短い | 1話として処理 |
| 500〜2000字 or 概念3〜4個 | 適切 | 1話として処理 |
| 2000字超 or 概念5個以上 or 明確な節の区切りがある | 長い | chapter-splitで分割 |

---

## ファイル保存ルール

**ファイル名：**
```
content/stories/[slug]-ch[XX].md

例：
content/stories/genri-kouron-ch01.md
content/stories/genri-kouron-ch02.md
```

slugは `--slug` オプションで指定。未指定の場合はAIが原典名から自動生成。
chは01, 02, 03... と連番。

**フロントマター（自動生成）：**
```markdown
---
title: "第X話：[タイトル]"
date: "YYYY-MM-DD（今日の日付）"
source: "[原典名]"
chapter: "[元の章・節]"
description: "[1〜2文の概要]"
---
```

---

## 確認が入るタイミング

chapter-splitで複数話に分割した場合のみ、以下を表示してから停止する：

```
📖 分割案：
  第1話：「タイトル」— [概要]
  第2話：「タイトル」— [概要]
  ...

このまま全話を物語化しますか？
「はい」または「進めてください」で続行します。
特定の話だけ処理したい場合は「第1話と第3話だけ」のように指定できます。
```

## 完了レポートのフォーマット

```
✅ 処理完了

生成したファイル：
  📄 content/stories/genri-kouron-ch01.md — 「第1話：タイトル」
  📄 content/stories/genri-kouron-ch02.md — 「第2話：タイトル」

次のステップ：
  git add content/stories/
  git commit -m "第X〜X話を追加"
  git push
```

## 処理詳細（各ステップの内部動作）

### STEP 1: concept-extract（内部実行）
- 主要概念をリストアップ
- 初心者の疑問を想定
- 比喩・たとえ話の候補を準備
- → 結果はSTEP 3のnarrativizeのプロンプトに自動注入

### STEP 2: 分割判断
- 文字数・概念数・節の区切りを総合判断
- 1話に収まる場合はSTEP 3へ直行
- 複数話に分割する場合はchapter-splitを実行し確認を取る

### STEP 3: narrativize（各話）
- キャラクター: 師匠（「〜じゃ」口調）、弟子（丁寧語）、ナレーター
- 舞台: 迷い人の茶屋
- テンプレートに従って対話形式で生成
- STEP 1の概念抽出結果を参照して比喩・たとえを選定

### STEP 4: story-review（内部実行）
チェック項目：
- 師匠・弟子のキャラ一貫性
- 原典の意味の正確さ
- 初心者への配慮（専門用語の補足）
- 物語の完成度
- → 問題があれば自動修正してからSTEP 5へ

### STEP 5: 自動保存
- `content/stories/[slug]-ch[XX].md` として保存
- 既存ファイルと重複する場合は `-v2` を付けて保存
