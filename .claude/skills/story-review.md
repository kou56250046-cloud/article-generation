# story-review — レビュー Skill

生成した物語の品質・一貫性・忠実度をチェックする。

## 使い方

```
/story-review
[レビューする物語のMarkdownを貼り付け]
```

---

## チェック項目と評価基準

### 1. キャラクター一貫性（師匠・弟子）
- [ ] 師匠の口調が「〜じゃ」「〜なのじゃ」で統一されているか
- [ ] 師匠がいきなり答えを言わず、問いや比喩から入っているか
- [ ] 弟子が最初から理解しすぎていないか（段階的な理解になっているか）
- [ ] 弟子の口調が丁寧語になっているか

### 2. 原典への忠実度
- [ ] 原典の主要概念が正しく伝わっているか
- [ ] 意味を歪めたり、本来ない概念を追加していないか
- [ ] 「この話の核心」が原典の本質を正しく一文で表しているか

### 3. 初心者への配慮
- [ ] 専門用語を使った場合、直後にたとえ話で補足しているか
- [ ] 一般の読者が理解できる表現になっているか
- [ ] 読了後に「なるほど」と感じられる構成か

### 4. 物語としての完成度
- [ ] ナレーターの描写で場面が浮かぶか
- [ ] 対話の流れが自然か（唐突な展開がないか）
- [ ] 弟子の気づき・成長が伝わるか
- [ ] 「迷い人の茶屋」の世界観と一致しているか

### 5. フロントマター
- [ ] title・date・source・chapter・description が正しく設定されているか

---

## 出力フォーマット

```
## レビュー結果

### 総合評価: [★★★★☆ など5段階]

### ✅ 良い点
- [点1]
- [点2]

### ⚠️ 改善点
- [改善提案1]
- [改善提案2]

### 🔧 修正案
[具体的な修正が必要な箇所があれば、修正後の文章も提示]
```
